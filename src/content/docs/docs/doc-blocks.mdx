Storybookの**Doc blocks**は、コンポーネントやプロジェクトのドキュメント作成を支援するために提供される一連のReactコンポーネントです。これらは主にMDXファイル内で使用され、ストーリーブックの情報をドキュメントページに簡単に組み込むことができます。

## 1. Doc Blocksの主な使用方法

1.  **MDXファイル内での使用**:
    最も一般的な方法です。`<Meta>`、`<Primary>`、`<Controls>` などのブロックをMDXファイルにインポートして配置することで、MarkdownとStorybookのインタラクティブな要素を組み合わせたカスタムドキュメントを作成します。

2.  **Autodocsのページテンプレートのカスタマイズ**:
    自動生成されるドキュメントページのテンプレートを上書きする際にも使用されます。<br />
    `<Title>`、`<Subtitle>`、`<Description>` などのブロックを組み合わせて、独自のドキュメント構造を定義できます。

## 2. 主要な Doc Blocks 一覧

| Doc&nbsp;Block&emsp;&emsp; | 機能 | 説明 |
| :--- | :--- | :--- |
| **`<Meta>`** | **ドキュメントの定義** | MDXファイルを特定のコンポーネントのストーリーに紐づけたり、サイドバーでのドキュメントの位置を制御したりするために使用されます。コンテンツ自体はレンダリングしません。 |
| **`<Primary>`** | **プライマリストーリー表示** | ストーリーファイルで最初に定義された（プライマリの）ストーリーを `<Story>` ブロックとして表示します。 |
| **`<Canvas>`** | **インタラクティブなプレビュー** | ストーリーを**ツールバー付きのラッパー**で表示し、ソースコードのスニペット（`<Source>`）を自動的に提供します。 |
| **`<Story>`** | **ストーリーのレンダリング** | CSFファイルで定義された特定のストーリーを、注釈（パラメーター、Argsなど）が適用された状態でレンダリングします。 |
| **`<Controls>`** | **動的な Args テーブル** | 特定のストーリーのArgs（プロパティ）を動的なテーブルとして表示し、ユーザーが値を変更してストーリーを操作できるようにします。 |
| **`<ArgTypes>`** | **静的な Args テーブル** | コンポーネントのインターフェースをドキュメント化するために、Argsの型（`argTypes`）を静的なテーブルで表示します。 |
| **`<Description>`** | **説明文の表示** | コンポーネントやストーリーの JSDoc コメントから取得した説明文を表示します。 |
| **`<Source>`** | **ソースコード表示** | ストーリーのソースコードのスニペットを直接レンダリングします。 |
| **`<Stories>`** | **全ストーリーの表示** | ストーリーファイル内の全ストーリーをコレクションとしてレンダリングします。 |
| **`<Markdown>`** | **Markdownの埋め込み** | プレーンな Markdown ファイル（例：`README.md`）をインポートして MDX ファイル内に含めることができます。 |
| **`<Title>`/`<Subtitle>`** | **見出しの表示** | ドキュメントエントリの主見出し、または二次的な見出しとして機能します。 |
| **`<ColorPalette>`/`<IconGallery>`** | **デザイン資産のドキュメント化** | プロジェクトで使用されている色やアイコンのギャラリーを簡単にドキュメント化できます。 |
| **`<Unstyled>`** | **デフォルトスタイルの無効化** | 内部のコンテンツに Storybook のデフォルトのスタイルが適用されないように設定します。 |

## 3. カスタマイズ

ほとんどの**Doc blocks**は、Storybookの `parameters` を介して、コンポーネントレベルまたはグローバルにカスタマイズ可能です。<br />
また、MDX内でブロックのPropsとして直接カスタマイズすることも可能です。
```mdx
<Controls exclude={['style']}>
```

<Aside type="caution" title="注意点">
  **Doc blocks**は**MDX**ファイル用に設計されており、ストーリーファイル（CSF）内では使用できません。
</Aside>
